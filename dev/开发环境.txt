【设计原则】
- 这个文件夹只放"环境"(编译器/运行时/工具链)，项目代码请另放别处。
- 全英文路径，不用盘符映射。换硬盘可直接整盘复制 D:\dev。

========================================================
  怎么用
========================================================

[普通编译] 直接开终端敲命令(已全在 PATH):
    clang++ a.cpp -o a.exe
    gcc a.c -o a.exe
    cargo new myapp  /  cargo build
    go run main.go
    node app.js
    uv run python app.py

[用 MSVC(cl) 或 编译 Qt] 必须先初始化 MSVC 环境:
    call "D:\dev\vs\VC\Auxiliary\Build\vcvars64.bat"
    cl /EHsc a.cpp
  (PowerShell 里直接敲 vcvars 即可, profile 里已内置该命令)

[Qt + CMake 工程] (推荐方式):
    call "D:\dev\vs\VC\Auxiliary\Build\vcvars64.bat"
    cmake -G Ninja -DCMAKE_CXX_COMPILER=cl -DCMAKE_PREFIX_PATH=D:/dev/qt/6.10.3/msvc2022_64 .
    ninja
  运行前把 Qt 的 bin 加进 PATH(以便找到 Qt6Core.dll 等):
    set PATH=D:\dev\qt\6.10.3\msvc2022_64\bin;%PATH%

[装 / 搜 工具]
    scoop search <名字>
    scoop install <名字>

[体检环境] 干活前 / 出问题时先跑:
    pwsh D:\dev\doctor.ps1          快速体检 (版本+路径, 约10秒)
    pwsh D:\dev\doctor.ps1 -Build   完整体检 (每条工具链真编译真运行, 约1分钟)

[升级所有工具] 不要直接 scoop update *, 用安全升级脚本:
    pwsh D:\dev\upgrade.ps1
    (自动: 版本快照存 D:\dotfiles\scoop\snapshots → 升级 → 清理 → 体检;
     升级坏了按快照回退: scoop install <app>@<旧版本>)

========================================================
  换新硬盘 / 重装系统 后
========================================================
  ★ 推荐方式: 配置已代码化, 直接用 dotfiles 仓库一键还原
     git clone https://github.com/5834716/dotfiles D:\dotfiles
     cd D:\dotfiles ; .\bootstrap.ps1
     (自动装 Scoop + 全部工具 + 恢复 PowerShell/Git/Terminal 配置)
     日常改了配置后: cd D:\dotfiles ; .\sync.ps1 然后 commit + push

  手动方式(整盘复制):
  1. 把 D:\dev 整盘复制到新机器同样位置 (D:\dev)。
  2. 设环境变量: SCOOP = D:\dev\scoop。
  3. 把 D:\dev\scoop\shims 及各 apps\*\current\bin 加进 PATH。
  4. VS / Qt 若复制后异常，按上面版本号到官网重装。
  5. WSL 复制后就地注册: wsl --import-in-place Ubuntu-24.04 D:\dev\wsl\ext4.vhdx
  6. 装 Cursor 编辑器(本机装在 D:\dev\cursor)。
  注: Windows SDK 装在 C 盘，复制 D:\dev 不含它，随 VS 生成工具一起重装即可。
  注: Scoop 工具清单 / Git 配置 / PowerShell profile 的最新版本都在 dotfiles 仓库里。
